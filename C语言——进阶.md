### 整型的表示方式

————整数在内存中储存的是二进制数的补码

正数的原，反，补码相同

负数:

原码：将数字直接转换为二进制数字    

反码：原码符号位不变，其他位次依次取反（负数）

补码：反码加一得 到补码（负数）



加法：数的原码相加；

减法：数的补码相加

```c
#include<stdio.h>
//体现整数在内存中储存的是二进制数的补码
int main()
{
	char a = -1;//默认为有符号数
    //整数-1的原码10000000 00000000 00000000 00000001
    //补码为11111111 11111111 11111111 11111111
    //则a地址储存的补码为11111111
	signed char b = -1;
    //同理，b地址储存的补码为11111111
	unsigned char c = -1;
    //同理，c地址储存的补码为11111111
	printf("a=%d,b=%d,c=%d\n",a,b,c);
    //字符用%d输出时，发生整型提升
    //有符号数用1来提升
    //a：11111111 11111111 11111111 11111111
    //b：11111111 11111111 11111111 11111111
    //对应的原码为10000000 00000000 00000000 00000001即-1
    //无符号数用0来提升（是否是有无符号数按照变量类型为准）
    //c；00000000 00000000 00000000 11111111正数
    //对应的原码为00000000 00000000 00000000 11111111即255
    return 0;
}
```

### 浮点型的表示方法

根据IEEE 754，浮点数的二进制表示形式：

(-1)^s * M      * 2^E——>只需要存储s(符号位),E(指数位),M(尾数位)

例：5.0——>101.0——>(-1)^0 * 1.01 * 2^2

0.5——>0.1——>(-1)^0 * 1.0 * 2^(-1)

在内存中，直接按照下面格式存储浮点数

32位的5.0：

0 10000001 01000000000000000000000——40 a0 00 00（十六进制表示）

#### 有效数字M

> IEEE 754 对有效数字 M 和指数 E，还有一些特别规定。前面说过，1≤M<2，也就是说，M 可以写成 1.xxxxxxx 的形式，其中 xxxxxxx 表示小数部分。
>
> IEEE 754 规定，在计算机内部保存 M 时，默认这个数的第一位总是 1，因此可以被舍去，只保存后面的 xxxxxxx 部分。比如保存 1.01 的时候，只保存 01，等到读取的时候，再把第一位的 1 加上去。这样做的目的，是节省 1 位有效数字。以 32 位浮点数为例，留给 M 只有 23 位，将第一位的 1 舍去以后，等于可以保存 24 位有效数字。

#### 指数 E

> 首先，E 为一个无符号整数（unsigned int）。这意味着，如果 E 为 8 位，它的取值范围为 0~255；如果 E 为 11 位，它的取值范围为 0~2047。但是，我们知道，科学计数法中的 E 是可以出现负数的，所以 IEEE 754 规定，存入内存时 E 的真实值必须再加上一个中间数，对于 8 位的 E，这个中间数是 127；对于 11 位的 E，这个中间数是 1023。比如，2^10 的 E 是 10，所以保存成 32 位浮点数时，必须保存成 10+127=137，即 10001001。
>
> ------
>
> E 不全为 0 或不全为 1
>
> 这时，浮点数就采用下面的规则表示，即指数 E 的计算值减去 127（或 1023），得到真实值，再将有效数字 M 前加上第一位的 1。比如：0.5（1/2）的二进制形式为 0.1，由于规定正数部分必须为 1，即将小数点右移 1 位，则为 1.0*2^(-1)，其阶码为 - 1+127=126，表示为 01111110，而尾数 1.0 去掉整数部分为 0，补齐 0 到 23 位 00000000000000000000000，则其二进制表示形式为:
>
> 0 01111110 00000000000000000000000
>
> ------
>
> E 全为 0
>
> 这时，浮点数的指数 E 等于 1-127（或者 1-1023）即为真实值，有效数字 M 不再加上第一位的 1，而是还原为 0.xxxxxx 的小数。这样做是为了表示 ±0，以及接近于 0 的很小的数字。
>
> ------
>
> E 全为 1
>
> 这时，如果有效数字 M 全为 0，表示 ± 无穷大（正负取决于符号位 s）；

```c
int main()
{
	int n = 9;//原码为00000000 00000000 00000000 00001001,补码相同
	float* p = &n;
	printf("n=%d\n", n);//正常输出9
	printf("*p=%lf\n", *p);
	//将9的补码视为浮点数的二进制存储
	//0 00000000 00000000000000000001001
	//M全零，无穷小
	//输出为0.000000
    
    int n = -9;//原码为10000000 00000000 00000000 00001001,
    //补码11111111 11111111 11111111 11110111
	float* p = &n;
	printf("n=%d\n", n);//正常输出9
	printf("*p=%lf\n", *p);
	//将9的补码视为浮点数的二进制存储
	//1 11111111 11111111111111111110111
	//E全一，M非全零
	//输出为-nan

	*p = 9.0;
    //按照浮点型储存方式，二进制表示：1001.0——》(-1)^0 *1.001*2^3
	//储存0 10000010 00100000000000000000000
	printf("n=%d\n", n);
    //按照整型输出，01000001000100000000000000000000为整数，原码相同
	//十进制输出为1091567616
	printf("*p=%lf\n", *p);//正常输出9.000000

	return 0;
}
```

#### 回调函数

> 回调函数就是一个通过函数指针调用的函数。如果你把函数的指针（地址）作为参数传递给另一个函数，当这个指针被用来调用其所指向的函数时，我们就说这是回调函数。回调函数不是由该函数的实现方直接调用，而是在特定的事件或条件发生时由另外的一方调用的，用于对该事件或条件进行响应。

#### qsort()函数

```c
void qsort(void *base, size_t nitems, size_t size, int (*compar)(const void *, const void *));
```

参数说明

1. **`base`**: 指向待排序数组的指针（`void*` 类型，支持任意数据类型（不能进行解引用操作））。
2. **`nitems`**: 数组中元素的个数（`size_t` 是无符号整数类型）。
3. **`size`**: 数组中每个元素的大小（以字节为单位，通常用 `sizeof` 获取）。
4. **`compar`**: 指向**比较函数**的指针，用于定义两个元素的比较规则。

使用qsort需要引用stdlib.h

```c
int compare_chars(const void* a, const void* b) {//比较字符
    return (int)(*(const char*)a - *(const char*)b);
}
int main()
{
	char arr[] = "abcdheb";
    int n = strlen(arr);//排除字符串末尾的'\0'

    qsort(arr, n, sizeof(arr[0]), compare_chars);
    printf("%s\n", arr);
	return 0;
}
```

```c
void bubble_sort(void* arr, int n, int weight, int(*cmpare)(const void*, const void*)) {
    //传入一个无类型数组，元素个数，类型的大小，自己编写的对应该类型的比较函数
	int i = 0;
	for (i = 0; i < n - 1; i++) {
		int j = 0;
		int flag = 1;//判断排序是否完成
        //一次排序过程，如果flag==0，说明不需要继续排序
		for (j = 0; j < n - 1 - i; j++) {
            //(char*)arr，将arr定义为char*类型，精准定位每一个字节
            //(char*)arr+j * weight，表示arr数组第j个元素在char*类型的地址
			if (cmpare((char*)arr + j * weight, (char*)arr + (j + 1) * weight) > 0) {
                //前一个元素大于后一个，发生交换函数
				swap((char*)arr + j * weight, (char*)arr + (j + 1) * weight, weight);
                //传入arr数组第j和第（j+1）个元素在char*类型的地址
				flag = 0;
			}

		}
		if (flag == 1) {
			break;
            //flag==0，说明不需要继续排序
		}
	}
}
void swap(void* a, void* b, int weight) {
	int i = 0;
	char* temp_a = (char*)a;
	char* temp_b = (char*)b;
    //用char*类型的指针接受传入元素的首地址
    //便于按照一个字节一个字节的复制元素
	for (i = 0; i < weight; i++) {
		char temp = *temp_a;
		*temp_a = *temp_b;
		*temp_b = temp;
		temp_a++;//移动到下一个字节的位置，继续复制
		temp_b++;
	}
}
```

### memory move

```c
void* my_memmove(void* dest, const void* src, size_t size) {
    void* ret = dest;
    // 用临时指针变量保存转换后的地址（左值，可自增）
    char* d = (char*)dest;
    const char* s = (const char*)src;

    if (s > d) {
        // 从后往前复制
        while (size--) {
            *(d + size) = *(s + size);
        }
    } else {
        // 从前往后复制
        while (size--) {
            *d = *s;
            d++;  // 对左值自增，合法
            s++;  // 对左值自增，合法
        }
    }
    return ret;
}
```

### 结构体

结构式一些值（可以是不同类型的）的集合，这些值称为成员变量。

#### 内存对齐

##### 内存对齐

1. 第一个成员在与结构体变量偏移量为 0 的地址处。
2. 其他成员变量要对齐到某个数字（对齐数）的整数倍的地址处。

**对齐数** = 编译器默认的一个对齐数 与 该成员大小的**较小值**。

- vs 中默认的值为 8，gcc编译器没有默认值

1. 结构体总大小为最大对齐数（每个成员变量都有一个对齐数）的整数倍。
2. 如果嵌套了结构体的情况，嵌套的结构体对齐到自己的最大对齐数的整数倍处，结构体的整体大小就是所有最大对齐数（含嵌套结构体的对齐数）的整数倍。

```c
struct s1 {//内存偏移量从 0 开始
	char a;//char的对齐数为1，偏移量为0
	char b;//char的对齐数为1，偏移量为1
	int n;
    //int的对齐数为4，前一个对齐数的字节空间已存在数据
    //储存在第二个对齐数的字节空间，偏移量为4
    //一到四之间的字节空间浪费了
    //最大对齐数为4
    //八个字节，满足为最大对齐数的整数倍
    //一个s1占八个字节
};
struct s2 {//内存偏移量从 0 开始
	char a;//char的对齐数为1，偏移量为0
	int n;
    //int的对齐数为4，前一个对齐数的字节空间已存在数据
    //储存在第二个对齐数的字节空间，偏移量为4
    //零到四之间的字节空间浪费了
	char b;//char的对齐数为1，偏移量为8
    //最大对齐数为4
    //九个字节，不满足为最大对齐数的整数倍，继续占用下一个对齐数的字节空间
    //一个s2占十二个字节
};
int main()
{
	struct s1 arr1 = { 0 };
	struct s2 arr2 = { 0 };
	printf("%d %d", sizeof(arr1), sizeof(arr2));
	//8 12
	return 0;
}//通过调整结构的成员变量的顺序，可以优化内存的占用
//由于结构体内存较大，函数调用时最好传址
```

### 位段

位段的声明与结构类似，但是有两点不同

1. 位段成员必须是整型变量。

2. 位段成员名后面有一个冒号和一个数字。

   (位段的宽度必须 ≤ 对应基础类型的总比特数)

位段——二进制位

```c
struct A
{
	int _a:2;//两个比特
	int _b:5;//五个比特
};
```



#### 位段的内存分配

1. 位段的成员可以是 `int`、`unsigned int`、`signed int` 或者是 `char`（属于整形家族）类型
2. 位段的空间上是按照需要以 4 个字节（`int`）或者 1 个字节（`char`）的方式来开辟的。
3. 位段涉及很多不确定因素，位段是不跨平台的（不同平台的int大小不同），注重可移植的程序应该避免使用位段。

```c
struct s//该位段按照四个字节开辟空间，32个比特
{
	int a : 5;
	int b : 6;
  	int c : 3;
    //前面占用14个比特,剩余18个比特
    int d : 19;//无法存放，重新开辟
    //共两个int，八个字节
};
struct s_//该位段按照一个字节开辟空间，8个比特
{
	char a : 5;//一个字节
	char b : 6;//一个字节
  	char c : 3;
    char d : 4;//共同储存在一个字节
};
int main()
{
	printf("%d\n", sizeof(struct s));//8
    printf("%d\n",sizeof(struct s_));//3
	return 0;
}
```

```c
int main()
{
	struct s_ s1 = { 0 };
	s1.a = 10;
	s1.b = 15;
	s1.c = 7;
	s1.d = 3;
    //s1的存储结果为
    //000 01010|00 001111|0 0011 111(从左到右使用，从低位到高位存储)
    //浪费 a    浪费 b    浪费 d  c
    //内存显示(十六进制)
    //0a 0f 1f
	return 0;
}
```

#### 位段的跨平台问题

1. int 位段被当成有符号数还是无符号数是不确定的。
2. 位段中最大位的数目不能确定。（16 位机器最大 16，32 位机器最大 32，写成 27，在 16 位机器会出问题。
3. 位段中的成员在内存中从左向右分配，还是从右向左分配标准尚未定义。
4. 当一个结构包含两个位段，第二个位段成员比较大，无法容纳于第一个位段剩余的位时，是舍弃剩余的位还是利用，这是不确定的。

总结：

跟结构相比，位段可以达到同样的效果，但是可以很好的节省空间，但是有跨平台的问题存在。

### 文件指针

**文件指针（FILE *）**

C 语言用`FILE`结构体表示文件，通过`FILE *`类型的指针操作文件。它是文件操作的 “句柄”，所有文件函数都需要通过它访问文件。

**文本文件 vs 二进制文件**

- **文本文件**：以 ASCII 码存储，人类可读（如`.txt`），换行符会自动转换（Windows 下`\r\n` ↔ `\n`）。
- **二进制文件**：直接存储数据的二进制形式（如`.exe`、图片），读写效率高，无换行符转换。

**流（Stream）**

C 语言将文件操作抽象为 “流”，分为：

- **标准流**：`stdin`（键盘输入）、`stdout`（屏幕输出）、`stderr`（错误输出）。
- **文件流**：通过`fopen`打开的文件流。

宏没有函数参数的调用和返回，比函数效率更高

> 用于调用函数和从函数返回的代码可能比实际执行这个小型计算工作所需要的时间更多。所以宏比函数在程序的规模和速度方面更胜一筹。
>
> 更为重要的是函数的参数必须声明为特定的类型。所以函数只能在类型合适的表达式上使用。反之这个宏怎可以适用于整形、长整型、浮点型等可以用于 > 来比较的类型。宏是类型无关的。

> 1. 每次使用宏的时候，一份宏定义的代码将插入到程序中。除非宏比较短，否则可能大幅度增加程序的长度。
> 2. 宏是没法调试的。
> 3. 宏由于类型无关，也就不够严谨。
> 4. 宏可能会带来运算符优先级的问题，导致程容易出现错。

###### 替换规则

> 在程序中扩展 #define 定义符号和宏时，需要涉及几个步骤。
>
> 1. 在调用宏时，首先对参数进行检查，看看是否包含任何由 #define 定义的符号。如果是，它们首先被替换。
> 2. 替换文本随后被插入到程序中原来文本的位置。对于宏，参数名被他们的值替换。
> 3. 最后，再次对结果文件进行扫描，看看它是否包含任何由 #define 定义的符号。如果是，就重复上述处理过程。
>
> 注意：
>
> 1. 宏参数和 #define 定义中可以出现其他 #define 定义的变量。但是对于宏，不能出现递归。
> 2. 当预处理器搜索 #define 定义的符号的时候，字符串常量的内容并不被搜索。

| **兼容性**   | 主流编译器（GCC/Clang/MSVC）全支持，但非 C/C++ 标准 | 是标准写法，**100% 兼容所有编译器**                |
